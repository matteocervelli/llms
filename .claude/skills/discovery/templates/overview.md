# Discovery — Phase 0 Overview

Validate before you build. Four modes, run in sequence:

| Mode          | Purpose                                               | Key Output                         |
| ------------- | ----------------------------------------------------- | ---------------------------------- |
| `problem`     | Who has this problem, how urgent, what's the evidence | Problem statement + user segment   |
| `competitive` | What already exists, why users would choose you       | Competitive map + differentiator   |
| `feasibility` | Can you actually build this, at what cost/time        | Feasibility verdict per dimension  |
| `checkpoint`  | Go / no-go gate before /story and /design             | Signed-off decision with rationale |

## Recommended Sequence

```
/discovery problem       → frame the problem
/discovery competitive   → map the landscape
/discovery feasibility   → check buildability
/discovery checkpoint    → get your verdict
```

Then: **Go** → `/story create` | **No-go** → revisit or pivot
