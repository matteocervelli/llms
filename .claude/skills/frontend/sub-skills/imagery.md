# Imagery & Visual Anchors

## Purpose

The `backgrounds.md` dimension covers gradients, textures, and atmospheric details.
This sub-skill covers **photographic imagery and visual anchor strategy** — how real images
do narrative work inside a page, especially in hero and section contexts.

## Core Principle: Imagery Must Do Narrative Work

An image that could be replaced by a gradient is not doing enough.
Every image on a page should show the product, place, atmosphere, or context — not decorate.

**Test for any image you're about to use:**

> "If I replace this image with a subtle gradient, does the user lose meaningful information
> about what this product or place is?"
>
> - No → the image is decorative. Use a real visual anchor instead.
> - Yes → the image is doing narrative work. Keep it.

## Hero Visual Anchor

The first viewport requires a strong visual anchor. "Atmospheric" is not enough.

- Show the product in use, the place itself, the people, or the context
- Prefer in-situ photography (product in environment) over abstract 3D objects
- Decorative gradients and geometric textures do not count as visual anchors

**Weak anchors** (reconsider): abstract gradients, geometric grids, floating UI mockups,
3D isometric renders, colorful blobs.

**Strong anchors**: product photography, venue/location photography, editorial portrait,
lifestyle imagery, real screenshots in context.

## Image Selection Rules

- **Tonal stability**: Choose or crop images with a calm, consistent tonal area where text
  will sit. Busy imagery behind text forces overlays that weaken both the image and the copy.
- **No embedded UI**: Do not use images with UI frames, split-screen panels, cards,
  or "app inside image" compositions — they fight the surrounding UI.
- **No embedded text**: Avoid images with signage, labels, or typographic elements
  that compete with your UI copy.
- **No collages**: Multiple moments need multiple images in a proper layout, not one collage.

## Mood Board Process

Before selecting final assets for a significant design, build a mood board first:

1. Describe the attributes the images should capture: style, color palette, composition, mood
2. Collect 4-6 references that share the same visual direction
3. Identify the common tonal area, composition pattern, and color temperature
4. Select final assets that fit within those parameters

Example description:

```
Style: editorial, slightly desaturated
Palette: warm neutrals, cream, aged paper tones
Composition: off-center subject, generous negative space right
Mood: quiet luxury, stillness, craftsmanship
```

## Handling Multiple Images

When a section shows multiple items (product shots, team photos, examples):

- Use the same lighting temperature across all images
- Apply consistent cropping ratio
- Do NOT use a collage — it loses individual impact and creates visual noise
- Consider a sequential layout (one image per beat of the section's narrative)

## Test: Is This Image Strong Enough?

Apply after placing any hero or section image:

> "If I remove this image and leave the text + layout, is the page worse?"
>
> - Barely → the image is too weak. Find a stronger visual anchor.
> - Yes, meaningfully worse → the image is pulling its weight.
