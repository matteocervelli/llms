# Competitive Landscape — Patterns (L2)

## Feature Matrix

Map competitors along dimensions that matter to your target segment:

| Feature / Capability      | You | Competitor A  | Competitor B | Do-nothing |
| ------------------------- | --- | ------------- | ------------ | ---------- |
| [Core job supported]      | ✓   | ✓             | Partial      | ✗          |
| [Segment-specific need]   | ✓   | ✗             | ✗            | Manual     |
| [Pricing tier accessible] | ✓   | ✗ (expensive) | ✓            | Free       |
| [Integration needed]      | ✓   | ✓             | ✗            | N/A        |

**Only include features that matter to your target segment.** A feature matrix with 20 rows usually signals unclear positioning.

## Moat Analysis

Ask: "If this works, what stops a well-funded competitor from copying it in 12 months?"

| Moat Type          | Strong if…                                            | Weak if…                                  |
| ------------------ | ----------------------------------------------------- | ----------------------------------------- |
| **Data**           | Proprietary dataset that improves with usage          | Data can be scraped or bought             |
| **Network**        | Value grows with each additional user                 | Value is the same with 1 user or 1M users |
| **Switching cost** | Users export their work or workflows into your system | Users can leave with one click            |
| **Brand**          | Users specifically seek you out by name               | Users found you via search/ads            |
| **Regulatory**     | Compliance certification takes years to obtain        | No regulatory differentiation             |
| **Speed**          | You can ship faster than incumbents structurally can  | Incumbent can hire 10 engineers tomorrow  |

**No moat = commodity.** That's okay if you're entering a large market with high switching cost, but name the risk.

## Positioning Dimensions

Pick **one primary differentiator**. You can have secondary ones, but lead with one:

- **Price**: You're accessible where alternatives are not (be careful — price is not a moat)
- **Segment focus**: You serve a specific niche that incumbents ignore
- **Speed**: You solve the problem 10× faster
- **Simplicity**: You do less, but the core job better (Apple vs. Sony)
- **Integration**: You connect things that existing tools don't

**The "too broad" test**: If your pitch works for 5 unrelated competitors, your positioning isn't specific enough.

---

## Why Users Leave Alternatives (Churn Drivers)

The best competitive insight comes from ex-customers, not current users:

Questions to ask (or find in reviews/forums):

- "What made you stop using [competitor]?"
- "What were you hoping [competitor] would do that it didn't?"
- "What did you switch to and why?"

**Common churn drivers**: Too complex, too expensive, doesn't integrate with X, missing one critical feature, poor support.

---

## Anti-Patterns

- **"We have no competition"**: Wrong. They're using spreadsheets, email, or nothing. Find it.
- **Feature parity as strategy**: Copying competitors' features doesn't differentiate you
- **Underestimating do-nothing**: Free + familiar beats paid + better for low-urgency problems
- **Positioning on price alone**: Unless you're Walmart, this is a race to the bottom
